# Function to loop for venues through all neighbourhoods
#url = 'https://api.foursquare.com/v2/venues/search?categoryId=50aa9e094b90af0d42d5de0d,530e33ccbcbc57f1066bbff3,530e33ccbcbc57f1066bbff9,4f2a25ac4b909258e854f55f&intent=browse&client_id={}&client_secret={}&v={}&ll={},{}&radius={}&limit={}'.format()
def getNearbyVenues(names, latitudes, longitudes, radius):
    venues_list=[]
    count = 0
    for name, lat, lng in zip(names, latitudes, longitudes):
        #print(name)
        
        # create the API request URL
        url = 'https://api.foursquare.com/v2/venues/explore?&client_id={}&client_secret={}&v={}&ll={},{}&radius={}&limit={}'.format(
            CLIENT_ID, 
            CLIENT_SECRET, 
            VERSION, 
            lat, 
            lng, 
            radius, 
            LIMIT)
        
        try:
          # make the GET request
          results = requests.get(url).json()["response"]['groups'][0]['items']
          #print(results)

          # return only relevant information for each nearby venue
          venues_list.append([(
              name, 
              lat, 
              lng, 
              v['venue']['name'], 
              v['venue']['location']['lat'], 
              v['venue']['location']['lng'],  
              v['venue']['categories'][0]['name']) for v in results])
        except:
            pass
        count = count+1
        if count == 700:
            break
        
    nearby_venues = pd.DataFrame([item for venue_list in venues_list for item in venue_list])
    nearby_venues.columns = ['Neighbourhood', 
                  'Neighbourhood Latitude', 
                  'Neighbourhood Longitude', 
                  'Venue', 
                  'Venue Latitude', 
                  'Venue Longitude', 
                  'Venue Category']
    
    return(nearby_venues)

Modelo 2: Gradient boosted decision trees¶

El refuerzo es una t√©cnica de conjunto donde se agregan nuevos modelos para corregir los errores cometidos por los modelos existentes. Los modelos se agregan secuencialmente hasta que no se puedan realizar m√°s mejoras. Un ejemplo popular es el algoritmo AdaBoost que pondera los puntos de datos que son dif√≠ciles de predecir.

El aumento de gradiente es un enfoque donde se crean nuevos modelos que predicen los residuos o errores de modelos anteriores y luego se suman para hacer la predicci√≥n final. Se llama aumento de gradiente porque utiliza un algoritmo de descenso de gradiente para minimizar la p√©rdida al agregar nuevos modelos.

XGBoost (eXtreme Gradient Boosting) es una implementaci√≥n de √°rboles de decisi√≥n potenciados por gradiente dise√±ados para la velocidad y el rendimiento. Es un algoritmo muy popular que recientemente ha dominado el aprendizaje autom√°tico aplicado para datos estructurados o tabulares.

Este enfoque admite problemas de modelado predictivo de regresi√≥n y clasificaci√≥n. Para obtener m√°s informaci√≥n, vea aqu√≠ para escuchar a Tianqi Chen, el creador de la biblioteca XGBoost.

Es muy probable que este modelo proporcione la mejor precisi√≥n posible y una medida de la importancia de la caracter√≠stica en comparaci√≥n con nuestra regresi√≥n hed√≥nica (aparte de los posibles peque√±os aumentos de precisi√≥n de la sintonizaci√≥n de hiperpar√°metros) debido al rendimiento superior de XGBoost.

Importancia de la característica¶

Importancia de la caracter√≠stica Adem√°s de su rendimiento superior, una ventaja de usar conjuntos de m√©todos de √°rbol de decisi√≥n como el aumento de gradiente es que pueden proporcionar autom√°ticamente estimaciones de la importancia de las caracter√≠sticas de un modelo predictivo entrenado.

En general, la importancia proporciona una puntuaci√≥n que indica cu√°n √∫til o valiosa fue cada caracter√≠stica en la construcci√≥n de los √°rboles de decisi√≥n potenciados dentro del modelo. Cuanto m√°s se utiliza un atributo para tomar decisiones clave con √°rboles de decisi√≥n, mayor es su importancia relativa.

Esta importancia se calcula expl√≠citamente para cada atributo en el conjunto de datos, lo que permite clasificar y comparar los atributos entre s√≠.

La importancia se calcula para un solo √°rbol de decisi√≥n por la cantidad que cada punto de divisi√≥n de atributo mejora la medida de rendimiento, ponderada por el n√∫mero de observaciones de las que es responsable el nodo. La medida del rendimiento puede ser la pureza (√≠ndice de Gini) utilizada para seleccionar los puntos de divisi√≥n u otra funci√≥n de error m√°s espec√≠fica.

Las caracter√≠sticas importantes se promedian en todos los √°rboles de decisi√≥n dentro del modelo.
