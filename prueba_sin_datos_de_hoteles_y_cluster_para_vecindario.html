El objetivo de mi trabajo es determinar si la presencia y las caractersticas de hoteles o negocios cercanos a una propiedad de alquiler turstico tiene un impacto en los precios de alquiler. Para llevar a cabo mi trabajo, he aunado datos de alquiler turstico de corta duracin (AirBnB) en la ciudad de Berln con datos de Foursquare sobre la ubicacin y caracterstica de locales de inters en la misma ciudad con datos de hoteles obtenidos en TripAdvisor. Luego he comparado los precios de alquiler con la presencia o ausencia de hoteles o negocios cercanos. Adems, tambin he considerado otros factores que podran afectar los precios de alquiler, como la ubicacin, el tamao de la propiedad y las caractersticas de los locales (si son restaurantes, tiendas por ejemplo o si un hotel tiene 1 o 4 estrellas). Para llevar a cabo mi trabajo, he utilizado un modelo de precios hednicos para determinar el impacto de la presencia de hoteles o negocios cercanos y otros factores en los precios de alquiler turstico. Inicialmente, he realizado una regresin lineal con las variables transformadas en logaritmos (el modelo tradicional de precios hednicos), pero el R^2 obtenido no fue satisfactorio. Por lo tanto, decid probar otros mtodos de regresin. En primer lugar, utilic una regresin con regularizacin ridge para ver si poda mejorar el ajuste del modelo. Sin embargo, an no obtuve resultados satisfactorios. Tambin lo hice con una regresin Lasso que tampoco me ha dado resultados significativos. Finalmente, utilic un rbol de decisin gradient xboost para ajustar el modelo y, finalmente, obtuvimos un R^2 aceptable y resultados significativos. Tambin he aplicado un rbol de decisin para ver si las variables seleccionadas eran determinantes en la determinacin de precio (utilizando anova en vez de class). En este sentido, he trabajado con modelos de aprendizaje supervisado como propuesto en el enunciado de la prctica.

#url = 'https://api.foursquare.com/v2/venues/search?categoryId=50aa9e094b90af0d42d5de0d,530e33ccbcbc57f1066bbff3,530e33ccbcbc57f1066bbff9,4f2a25ac4b909258e854f55f&intent=browse&client_id={}&client_secret={}&v={}&ll={},{}&radius={}&limit={}'.format()
def getNearbyVenues(names, latitudes, longitudes, radius):
    venues_list=[]
    count = 0
    for name, lat, lng in zip(names, latitudes, longitudes):
        #print(name)
        
        # create the API request URL
        url = 'https://api.foursquare.com/v2/venues/explore?&client_id={}&client_secret={}&v={}&ll={},{}&radius={}&limit={}'.format(
            CLIENT_ID, 
            CLIENT_SECRET, 
            VERSION, 
            lat, 
            lng, 
            radius, 
            LIMIT)
        
        try:
          # make the GET request
          results = requests.get(url).json()["response"]['groups'][0]['items']
          #print(results)

          # return only relevant information for each nearby venue
          venues_list.append([(
              name, 
              lat, 
              lng, 
              v['venue']['name'], 
              v['venue']['location']['lat'], 
              v['venue']['location']['lng'],  
              v['venue']['categories'][0]['name']) for v in results])
        except:
            pass
        count = count+1
        if count == 700:
            break
        
    nearby_venues = pd.DataFrame([item for venue_list in venues_list for item in venue_list])
    nearby_venues.columns = ['Neighbourhood', 
                  'Neighbourhood Latitude', 
                  'Neighbourhood Longitude', 
                  'Venue', 
                  'Venue Latitude', 
                  'Venue Longitude', 
                  'Venue Category']
    
    return(nearby_venues)

def return_most_common_venues(row, num_top_venues):
    row_categories = row.iloc[1:]
    row_categories_sorted = row_categories.sort_values(ascending=False)
    
    return row_categories_sorted.index.values[0:num_top_venues]

Modelo 1: Spatial Hedonic Price Model (HPM)¶

El modelo hednico implica la regresin de los precios de venta observados en el listing en comparacin con los atributos de una propiedad hipottica que determina el precio de venta. Proviene de la teora hednica de precios que supone que una mercanca, como una casa, puede verse como una agregacin de componentes o atributos individuales (Griliches, 1971). Se supone que los consumidores compran bienes que incorporan paquetes de atributos que maximizan sus funciones de utilidad subyacentes (Rosen, 1974).

Adems de las caractersticas de los listados de Airbnb, agregamos caractersticas de ubicacin, ya que se ha demostrado que son factores importantes para influir en el precio. Idealmente, las pruebas de multiplicador de Lagrange deben realizarse para verificar si hay un retraso espacial en la variable dependiente y, por lo tanto, se prefiere un modelo de retraso espacial para estimar un HPM espacial. Sin embargo, para los fines de este modelo solo estamos utilizando un modelo OLS convencional para la estimacin de precios hednicos que incluye caractersticas espaciales y de ubicacin, pero no un retraso espacial que explica la dependencia espacial.

Por lo tanto, las primeras variables explicativas son las caractersticas de los listados (alojamientos, baos, etc.) y nuestro segundo grupo de variables explicativas basadas en caractersticas espaciales y de ubicacin son Score, que es la distancia de red al quinto lugar ms cercano que calculamos con Pandana; y Barrio perteneciente, 1 si el listado pertenece al barrio especificado, 0 en caso contrario.

Modelo 2: Gradient boosted decision trees¶

El refuerzo es una tcnica de conjunto donde se agregan nuevos modelos para corregir los errores cometidos por los modelos existentes. Los modelos se agregan secuencialmente hasta que no se puedan realizar ms mejoras. Un ejemplo popular es el algoritmo AdaBoost que pondera los puntos de datos que son difciles de predecir.

El aumento de gradiente es un enfoque donde se crean nuevos modelos que predicen los residuos o errores de modelos anteriores y luego se suman para hacer la prediccin final. Se llama aumento de gradiente porque utiliza un algoritmo de descenso de gradiente para minimizar la prdida al agregar nuevos modelos.

XGBoost (eXtreme Gradient Boosting) es una implementacin de rboles de decisin potenciados por gradiente diseados para la velocidad y el rendimiento. Es un algoritmo muy popular que recientemente ha dominado el aprendizaje automtico aplicado para datos estructurados o tabulares.

Este enfoque admite problemas de modelado predictivo de regresin y clasificacin. Para obtener ms informacin, vea aqu para escuchar a Tianqi Chen, el creador de la biblioteca XGBoost.

Esta importancia se calcula explcitamente para cada atributo en el conjunto de datos, lo que permite clasificar y comparar los atributos entre s.

La importancia se calcula para un solo rbol de decisin por la cantidad que cada punto de divisin de atributo mejora la medida de rendimiento, ponderada por el nmero de observaciones de las que es responsable el nodo. La medida del rendimiento puede ser la pureza (ndice de Gini) utilizada para seleccionar los puntos de divisin u otra funcin de error ms especfica.

Las caractersticas importantes se promedian en todos los rboles de decisin dentro del modelo.
